## Теория

### Сортировка пузырьком (Bubble Sort)

Алгоритм последовательно сравнивает соседние элементы и меняет их местами, если они стоят в неправильном порядке. Проход повторяется до тех пор, пока массив не будет отсортирован. Используется оптимизация с флагом — если за проход не было ни одного обмена, массив уже отсортирован и алгоритм завершается досрочно.

**Сложность:**

| Случай | Файл в программе | Сложность |
|---|---|---|
| Лучший (уже отсортирован) | `direct_order.txt` | O(n) — один проход без обменов |
| Худший (обратный порядок) | `reverse_order.txt` | O(n²) — максимум сравнений и обменов |
| Средний (случайный порядок) | `random_order.txt` | O(n²) |

- Сложность по памяти: O(1) — сортировка на месте.
- Алгоритм **устойчивый** — не меняет порядок элементов с одинаковыми ключами.

### Бинарный поиск

Работает только на отсортированном массиве. Делит массив пополам и сравнивает средний элемент с искомым ключом. Сложность: **O(log n)**.

## Структура проекта

- `struct.h` — описание структуры `Item` (поля `key`, `value`).
- `bubble_sort.c` / `bubble_sort.h` — сортировка пузырьком.
- `binary_search.c` / `binary_search.h` — рекурсивный бинарный поиск по ключу.
- `read_file.c` / `read_file.h` — чтение данных из файла:
    - `get_list_size` — получение количества элементов из первой строки файла
    - `read_file` — чтение массива элементов из файла
- `cli.c` / `cli.h` — интерфейс командной строки.
- `main.c` — точка входа, запуск CLI.
- `tests.c` — 5 тестов через `assert`.
