## Вариант

**Структура данных:** АВЛ-дерево (самобалансирующееся бинарное дерево поиска).

- Реализованы левый и правый повороты, а также балансировка (в т.ч. двойные повороты: Left-Right и Right-Left).
- Поддерживается загрузка команд из файла (`files/input.txt`) и интерактивная работа через консоль.
- Реализована функция освобождения памяти (`free_tree`).

## Объяснение алгоритмов

### Балансировочный фактор (bfactor)
```
bfactor(node) = height(right) - height(left)
```
Узел сбалансирован, если `bfactor ∈ {-1, 0, 1}`.

### Повороты

**Правый поворот** (Right Rotation) — применяется, когда левое поддерево перевешивает:
```
    y              x
   / \    --->   /   \
  x   C         A     y
 / \                  / \
A   B                B   C
```

**Левый поворот** (Left Rotation) — зеркально, когда правое поддерево перевешивает.

### Двойные повороты

| Случай | bfactor(node) | bfactor(child) | Действие                                   |
|---|---|----------------|--------------------------------------------|
| Left-Left | -2 | <= 0           | Правый поворот                             |
| Left-Right | -2 | > 0            | Левый поворот child -> Правый поворот node |
| Right-Right | +2 | >= 0           | Левый поворот                              |
| Right-Left | +2 | < 0            | Правый поворот child -> Левый поворот node |

### Удаление узла
При удалении узла с двумя детьми узел заменяется **минимальным элементом правого поддерева** (`find_min`), после чего этот минимум удаляется из правого поддерева (`remove_min`), и выполняется перебалансировка.

### Формат входного файла (`input.txt`)
```
1 ключ значение   # вставка
2 ключ            # удаление
3                 # печать дерева
4 ключ            # поиск
0                 # выход
```

## Вывод

В ходе лабораторной работы реализовано АВЛ-дерево на языке C с поддержкой строковых ключей и вещественных значений. Дерево поддерживает все базовые операции: вставку, удаление, поиск и печать с сохранением инварианта балансировки (|bfactor| ≤ 1) после каждой операции.

Гарантированная высота АВЛ-дерева - **O(log n)**, что обеспечивает логарифмическое время для всех операций в отличие от несбалансированного BST, который в худшем случае деградирует до связного списка (O(n)).
