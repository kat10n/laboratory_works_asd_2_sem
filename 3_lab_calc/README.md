## Теория

### Обратная польская нотация (Reverse Polish Notation)

Алгоритм Дейкстры переводит инфиксное выражение в постфиксную запись, в которой операторы следуют после своих операндов. Используется стек для хранения операторов с учётом их приоритета.

**Приоритеты операторов:**

| Приоритет | Операторы |
|-----------|-----------|
| 0 | `+`, `-` |
| 1 | `*`, `/`, `%` |
| 2 | `^` |

### Дерево выражений

По строке в ОПН строится бинарное дерево: операнды становятся листьями, операторы — внутренними узлами. Левый потомок — первый операнд, правый — второй.

### Упрощение выражения по дереву

После построения дерева программа рекурсивно преобразует только операции умножения:
- `a * (b + c)` → `(a * b) + (a * c)`
- `(a + b) * c` → `(a * c) + (b * c)`
- `7 * 6 * j * k * 4` → `168 * j * k`

Сложение, вычитание и деление не вычисляются. Программа считает только числовые множители.

## Структура проекта

- `struct.h` — структуры `Stack`, `Stack2`, `Node`.
- `stack.c` / `stack.h` — реализация двух стеков: для символов (`Stack`) и для узлов дерева (`Stack2`).
- `enter.c` / `enter.h` — ввод строки и валидация выражения.
- `tokenize.c` / `tokenize.h` — разбиение строки на токены.
- `reverse_polish_notation.c` / `reverse_polish_notation.h` — перевод в ОПН, построение дерева выражений, вывод дерева.
- `mini_calculator.c` / `mini_calculator.h` — копирование дерева, упрощение умножения, вывод выражения из дерева.
- `main.c` — точка входа.
- `tests.c` — тесты через `assert`.

## Тесты

В `tests.c` реализованы проверки через `assert`:

1. **RPN** — перевод выражений в ОПН.
2. **Validation** — корректные и некорректные выражения.
3. **Multiply numbers** — вычисление всех числовых множителей в цепочке умножений.
4. **Distribution and multiply** — раскрытие умножения по `+` и `-` с последующим вычислением чисел.
